# Learning Roadmaps

A skill roadmap should start with the smallest project that proves the target behavior. For React, build a typed data dashboard with filters, loading states, and tests. For backend APIs, build authentication-free endpoints first, then add validation, caching, rate limiting, and integration tests. For AI engineering, build retrieval first, evaluate retrieved context, and only then add generation.

## Project-Based Learning

The fastest path to proof is to ship in layers. Week one should produce a running feature. Week two should add tests and documentation. Week three should add polish, deployment, and measurable outcomes. Avoid passive tutorial loops; after each tutorial, create a variant that changes requirements enough to force independent thinking.

## Testing Practice

Testing should match risk. UI flows need smoke tests for navigation and critical states. Backend services need mocked external API failures, empty input tests, and validation tests. LLM features need schema validation, malformed output handling, and fallback behavior that tells the user what happened.

## Skill Building Steps

For every missing skill, define a tiny proof artifact: a repository, a deployed endpoint, a benchmark, a demo video, or a README decision log. The artifact should answer three questions: what problem was solved, what tradeoff was made, and how the candidate knows it works.

# AI Engineering Learning

A practical AI engineering roadmap should include structured prompting, embeddings, vector search, retrieval evaluation, hallucination controls, and cost/rate-limit handling. A strong project shows source citations, handles empty retrieval, validates model output, and separates orchestration logic from API routes.
