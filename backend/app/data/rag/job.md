# Role Expectations

Software engineering roles usually expect evidence across implementation, debugging, testing, and communication. A candidate should be able to turn vague requirements into scoped work, explain tradeoffs, and demonstrate that the feature works. Hiring teams look for proof that the candidate can maintain code after the first demo.

## Frontend Engineer Skills

Frontend roles often require React, TypeScript, state management, API integration, accessibility, responsive design, and testing. Strong candidates show polished interfaces, predictable loading and error states, and attention to usability. Evidence is strongest when a deployed UI connects to real backend data.

## Backend Engineer Skills

Backend roles often require API design, validation, databases, caching, auth concepts, asynchronous jobs, observability, and tests. Strong candidates show clean service boundaries, safe error handling, rate-limit awareness, and realistic external API failure handling.

## AI Engineer Skills

AI engineering roles expect Python, LLM APIs, embeddings, RAG, evaluation, prompt design, data handling, and production reliability. A strong portfolio project uses real retrieval, cites sources, validates structured outputs, and handles model or provider failure without exposing secrets.

## Hiring Signals

Hiring teams reward evidence-backed depth. Strong signals include recent commits, concise READMEs, tests, deployment links, clear architecture diagrams, and code that can be reviewed quickly. Weak signals include long unverified skill lists, broken demos, missing setup instructions, and no explanation of tradeoffs.
